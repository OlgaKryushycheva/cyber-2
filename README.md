# Лабораторна робота №2: реалізація RSA на Java

Цей застосунок демонструє повний цикл роботи алгоритму RSA: генерацію ключів, шифрування та розшифрування. Інтерфейс створено на Swing і розрахований на швидку перевірку роботи алгоритму.

## Структура проєкту
- `src/main/java/org/example/RSAApp.java` — графічний інтерфейс із полями для ключів, відкритого тексту та шифртексту.
- `src/main/java/org/example/RSAService.java` — генерує ключі та виконує шифрування/розшифрування на основі `BigInteger`.
- `src/main/java/org/example/RSAKeyPair.java` — компактна модель пари ключів RSA.

## Як запустити
1. Переконайтеся, що встановлено JDK 17+ (проєкт перевірений на JDK 25).
2. Скомпілюйте код:
   ```bash
   javac src/main/java/org/example/*.java
   ```
3. Запустіть застосунок:
   ```bash
   java -cp src/main/java org.example.RSAApp
   ```

## Логіка алгоритму
1. **Генерація простих чисел**: дві великі ймовірно прості величини `p` та `q` з використанням `SecureRandom` і `BigInteger.probablePrime`.
2. **Обчислення модуля та функції Ейлера**: `n = p*q`, `phi = (p-1)*(q-1)`.
3. **Вибір відкритого показника**: стандартне значення `e = 65537`, при конфлікті підбирається найближчий взаємно простий із `phi` показник.
4. **Обчислення секретного показника**: `d = e^(-1) mod phi` через `modInverse`.
5. **Шифрування/розшифрування**: перетворення повідомлення у `BigInteger` (UTF-8), піднесення до степеня `e` або `d` за модулем `n`, представлення шифртексту у Base64.

## Поради з використання
- Для навчальних цілей достатньо ключів 1024–2048 біт. Занадто великі ключі уповільнять GUI.
- Якщо повідомлення занадто довге для вибраної довжини ключа, застосунок підкаже про необхідність збільшити розмір ключа.
- Шифртекст подається у Base64, тому легко копіюється та вставляється.
